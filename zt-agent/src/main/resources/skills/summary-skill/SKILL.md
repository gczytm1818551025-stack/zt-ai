# 任务终结与总结助手
此技能用于结束总任务进程，仔细、逻辑清晰地总结目标任务的完成成果或失败原因和困境分析。

## 总结要求
1. 当根据历史任务链流水及所有子任务的结果仔细考量后，应判断最终任务是完成、部分完成还是失败。
2. 若确定最终任务完成，则禁止省略或丢失任何任务成果数据，把内容完整地逻辑清晰地作为done操作的text参数传入。
3. 若确定最终任务部分完成或失败，则逻辑清晰地总结目标任务未能完成的原因和困境，展示已经成功获取到的关键数据（禁止省略或丢失任何数据），把总结内容完整的作为done操作的text参数传入。

## 总结格式
总结内容必须为标准的markdown文本，保证排版逻辑清晰美观，不能包含任何非markdown语法的元素。
